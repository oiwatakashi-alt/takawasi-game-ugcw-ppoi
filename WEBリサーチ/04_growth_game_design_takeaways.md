# 育成ゲームとしての面白さ抽出

調査日: 2026-06-30

## 結論

`Ultimate General: Civil War` の育成の面白さは、部隊を強くすること自体ではなく、強くなった部隊を「いつ使い、どこまで失ってよいか」をプレイヤーに悩ませるところにある。

育成リソースが戦闘結果で傷つき、戦闘結果を育成で回復する。この相互作用が強い。

## 面白さの型

### 1. 育成対象が複数ある

本作は、育てるものが1つではない。

- 部隊を育てる。
- 将校を育てる。
- 軍全体能力を育てる。
- 武器在庫を育てる。
- 評判を保つ。
- 補給体制を整える。

この多層育成があるため、単純な「経験値を稼いで強くなる」よりも深い。

### 2. 育成と損耗が同じ画面で起きる

戦闘中に育った部隊は、同じ戦闘中に死ぬ。将校も同じ。これが緊張感を生む。

育成ゲームの欠点は、強くなると作業になること。本作は、強い部隊ほど失いたくないため、むしろ操作が慎重になる。

### 3. 安い回復と高い回復がある

Rookie補充は安いが能力が落ちる。Veteran補充は高いが能力を保つ。

これが非常に重要。回復に質の差をつけることで、プレイヤーは「誰を本当に大事にするか」を選ぶ。

### 4. 成長を集中する価値がある

3星旅団、優秀な将校、高級武器を揃えると、明確な切り札になる。

ただし、切り札は維持費が高く、失うと痛い。集中育成には快感と不安が同居する。

### 5. 量の拡大にもリスクがある

Army Organizationを上げると軍が大きくなる。大軍は安心だが、補給、武器、将校、管理負荷、敵スケールの問題が増える。

普通の育成ゲームでは「枠拡張」は純粋な強化になりがち。本作では「大きくした軍を本当に維持できるか」が問われる。

### 6. 戦闘前準備が楽しい

Campでの準備は、見方によっては作業だが、本作ではかなりゲーム性がある。

- 次戦の地形/規模を見る。
- どの軍団を出すか決める。
- どの部隊にベテランを補充するか決める。
- どの部隊に良い武器を渡すか決める。
- 将校を入れ替える。
- 補給を買う。
- 評判を使うか温存する。

この準備フェーズが「育成ゲームの本編」として成立している。

## takawasi game への抽出案

### 最小プロトタイプ案

まずは以下だけでよい。

1. 3部隊だけ持つ。
2. 各部隊に `人数`, `経験`, `士気`, `疲労`, `指揮官` を持たせる。
3. 戦闘で人数が減り、経験が増える。
4. 戦闘後に `新人補充` と `熟練補充` を選ぶ。
5. 新人補充は安いが経験/士気が薄まる。
6. 熟練補充は高いが能力を維持する。
7. 指揮官は成長するが、一定確率で負傷/戦死する。
8. 次の戦闘に状態を持ち越す。

これで「育成した部隊を使う痛み」が出る。

### 戦闘側の最小要素

- 地形: 開地、森、高地、橋
- 命令: 前進、防衛、側面、休息
- 状態: 士気、疲労、補給
- 勝利条件: 拠点占領、敵士気崩壊、時間まで防衛

複雑なRTS操作を作る前に、部隊状態の持ち越しを先に作るべき。

### 育成UIの最小要素

戦闘後に1画面だけ用意する。

- 損耗結果
- 経験増加
- 負傷/戦死
- 補充選択
- 装備選択
- 次戦予告

ここでプレイヤーに迷わせる。

## 真似しない方がいい点

- 初期から軍団/師団/旅団の大規模階層を全部作る。
- 武器種類を大量に作る。
- 史実戦闘を細かく再現する。
- 大量ユニットRTSを先に作る。
- 補給、士気、疲労、将校、武器、地形を全部同時に詳細化する。

本作の表面は巨大だが、面白さの中心は「育ったものが戦闘で傷つき、次に残る」こと。まずそこだけ作る方がよい。

## 参考になる数値設計

本作から抽出するなら、数値は以下の方向がよい。

- 経験は戦闘参加と成果で上がる。
- 人数補充は能力平均を下げる。
- 熟練補充は高コスト。
- 指揮官ランクは部隊上限や効率に関係する。
- 士気は戦闘中の短期状態であり、評判/勝敗/損耗で長期補正を受ける。
- 地形はダメージ軽減だけでなく、視界、移動、隠密、命中に関わる。

## 企画上のキーワード

- 育成したいが、使うと壊れる。
- 勝ったのに失敗した気分が残る。
- 負けたのに精鋭を守れて次に希望が残る。
- 数字ではなく部隊名に愛着が湧く。
- 戦闘後の修理/補充画面が本編になる。
- 強化は直線ではなく、維持費とリスクを伴う。
