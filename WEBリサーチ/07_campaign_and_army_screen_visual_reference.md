# Campaign / Army Screen Visual Reference - Ultimate General: Civil War

調査日: 2026-06-30

## 見たもの

UGCWの公式ページ、Steam/コミュニティ/PlayStation系スクリーンショット、公式ゲームガイドPDFのCampaign / Army Management / Armory / Barracks / Career / Battle Map項目を確認した。

画像自体は再配布せず、画面設計に使う観察だけを残す。

## UGCWの画面レイヤー

UGCWは大きく分けると以下の構造。

```text
Campaign / Battle Map
  -> Camp
      -> Army
      -> Armory
      -> Barracks
      -> Career
  -> Deployment
  -> Battle
  -> Result
```

重要なのは、戦闘画面だけでゲームが成立しているのではなく、CampとBattle Mapが「育成ゲーム」の本体に近いこと。

## Campaign / Battle Map

UGCWのBattle Mapは、固定歴史キャンペーンの段階ごとに大戦闘と小戦闘を表示する画面。

見える/扱う要素:

- campaign stage name
- 次の大戦闘
- 任意の小戦闘
- 勝利/引き分け/敗北時の報酬
- 金、兵員、評判
- 部隊配置スロット
- 偵察レベルに応じた敵情報
- deploymentへの導線

UGCWでは小戦闘が大戦闘準備に効く。小戦闘で軍や評判を育て、敵規模や敵訓練に影響する場合もある。

## 本作側への変換

本作は史実キャンペーン固定ではないので、UGCWのBattle Mapをそのまま真似ない。

代わりに `Theater Command` 画面にする。

```text
Theater Command
  -> 5層戦略マップ
  -> プレイヤー軍現在位置
  -> 今ターンの主戦場
  -> 3-4個の小任務
  -> 敵圧/補給/施設/援軍/撤退路
```

UGCWの「大戦闘 + 小戦闘」を、本作では「スキップ不可の主戦場 + 自動解決可能な小任務」に変換する。

## Army Screen

UGCWのArmy画面は、Corps / Division / Brigadeの階層と、部隊詳細を管理する画面。

扱う要素:

- 軍団/師団/旅団の階層
- 旅団の作成
- 師団/軍団の作成
- 部隊の移動
- 部隊名変更
- 部隊解散
- 兵数補充
- Rookie / Veteran補充
- 武器変更
- 将校割当
- 部隊履歴
- 星/Perk

画面の本質:

- プレイヤーが「自分の軍」を読む場所。
- 数字だけでなく、名前・将校・歴戦・損耗が見える必要がある。
- 戦闘後の悩みを生む場所。

## 自作側のArmy Camp

本作では `Army Camp` をより大きくする。

必要な区分:

| 区分 | 内容 |
| --- | --- |
| Army Structure | 軍団/師団/旅団、出撃割当、予備部隊。 |
| Unit Detail | 兵数、経験、士気、疲労、弾薬、武器、将校、履歴、勲章。 |
| Replenishment | Rookie補充、Veteran補充、補充しない、部隊統合。 |
| Recovery | 休養、疲労回復、負傷者復帰、野戦病院効果。 |
| Deployment Readiness | 次の主戦場/小任務に出せる状態か。 |

UGCWのArmyタブに、本作では疲労、施設依存、戦略マップ位置、医療回復を足す。

## Armory Screen

UGCWのArmoryは、ShopとArmoryの左右分割が基本。

観察:

- 兵科タブで武器カテゴリを切り替える。
- 左に購入可能なShop。
- 右に保有しているArmory。
- 武器ごとに価格、在庫、性能バーがある。
- 装備変更は部隊補充と直結する。

本作側では `Armory / Logistics` に拡張する。

必要な要素:

- 武器在庫
- 弾薬在庫
- 補給物資
- 建設材料
- 砲弾/特殊弾薬
- 鉄道/輸送容量
- 敵からの鹵獲品
- 補給拠点への配分
- 小任務への資源割当

UGCWは武器管理が中心。本作は武器に加えて、防衛戦なので補給・弾薬・材料の比重が大きい。

## Barracks / Officers Screen

UGCWのBarracksは、将校の雇用/予備/負傷管理を扱う。

扱う要素:

- Academyの雇用候補
- Reserveの未配属将校
- 負傷将校
- 階級
- 経験
- 部隊/師団/軍団への割当
- 戦死/負傷による穴埋め

本作側では `Officers` 画面として独立させるべき。

必要な要素:

- 将校名
- 階級
- 所属
- 指揮容量
- 特性
- 戦歴
- 負傷/回復ターン
- 戦死記録
- 防衛戦向き/攻勢向き/工兵向き/兵站向き
- 昇進候補
- 小任務の自動解決に割り当て中か

将校は単なる能力値ではなく、軍全体の制約と愛着の源にする。

## Career Screen

UGCWのCareerはキャリアポイントと将軍能力を扱う。

代表カテゴリ:

- Politics
- Economy
- Medicine
- Training
- Army Organization
- Logistics
- Reconnaissance

本作側では `Doctrine / Staff` 画面にする。

UGCW相当:

- 政治/評判
- 経済
- 医療
- 訓練
- 軍編成
- 兵站
- 偵察

本作追加:

- 工兵
- 要塞構築
- 後退戦
- 鉄道補給
- 対アンデッド戦術
- 恐怖/士気対策
- 夜戦対策
- 冬季戦/DLC向け枝

## UIトーン

UGCWのキャンプ系UIは暗い背景に、茶/金系のフレーム、タブ、リスト、性能バーで構成される。

良い点:

- タブ構成が明確。
- 情報量は多いが、軍編成/武器/将校/キャリアで分かれている。
- 数値管理と軍事雰囲気が両立している。

本作で真似るべき点:

- 画面下部または上部に常時主要タブ。
- 左に一覧、右に詳細。
- 資源/評判/現在戦線は常時見える。
- 部隊名、将校名、戦歴を見せて愛着を作る。
- 1画面に詰め込みすぎず、戦略マップ、軍、将校、装備、工兵を分ける。

## 実装上の注意

- UGCWのCampタブを参考にするが、そのまま1画面に押し込まない。
- 本作は戦略キャンペーン画面が重要なので、Army Campとは分ける。
- 軍編成画面は後で軍団/師団/旅団が増えても崩れないツリー構造にする。
- 小任務自動解決の割当状態をArmy/Officers側にも表示する。
- Engineering Worksは最初から独立ドメインとして扱う。
- 資源バーは画面を跨いで一貫させる。
