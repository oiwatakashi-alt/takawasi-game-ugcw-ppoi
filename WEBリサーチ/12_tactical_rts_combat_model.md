# Tactical RTS Combat Model Note

Date: 2026-06-30

## 目的

UGCWの戦術戦闘画面をそのまま複製するのではなく、抽象化しすぎない最低限のRTS戦闘モデルとして `takawasi game` に落とす。

今回の焦点:

- 戦術マップ上で旅団、敵部隊、防衛施設が座標を持つ。
- 射程差を持つ武器が、射程内の最寄り目標を選んで撃つ。
- 敵は単なる数値圧ではなく、波ごとの部隊として前線へ移動する。
- 防衛施設は背景飾りではなく、遮蔽、減速、施設火力、耐久損耗を持つ。
- 編成/出撃配置で選んだ旅団だけが戦闘に入る。

## 参照したUGCW要素

公式ガイドPDFと戦闘基本情報から見ると、UGCW戦闘画面の面白さは「地形情報を読んで、旅団単位に命令し、士気/疲労/補給/射線/側面を管理する」点にある。

`takawasi game` の初期実装では、以下を優先して抽出する。

| UGCW側の観察 | takawasi gameでの初期実装 |
| --- | --- |
| 旅団単位で戦術マップに配置される | `BattleUnit.position` を持つ盤面駒として表示 |
| 武器と兵科で役割が違う | 武器ごとに `range`, `firepower`, `fireRate` を設定 |
| 敵軍も戦場上で動く | 敵波を `EnemyBattleUnit` として生成し、東側から前線へ移動 |
| 地形/防御線が戦闘判断に影響する | 戦術ノード、地形コールアウト、防衛施設、前線を表示 |
| 補給/士気/疲労が戦闘継続力になる | 弾薬、士気、conditionをtickで更新 |
| 防御施設やカバーが重要 | trench/barricade/supply depotが遮蔽、減速、火力、耐久を持つ |

## 採用モデル

1 tick = 1秒。

tickの順序:

```text
敵波スポーン
  -> 敵部隊移動
  -> 自軍命令に応じた移動
  -> 自軍が射程内の最寄り敵を射撃
  -> 防衛施設が射程内の最寄り敵を射撃
  -> 敵部隊が射程内の最寄り自軍を攻撃
  -> 敵接近で防衛施設が損耗
  -> 工兵が近距離施設を修理
  -> 士気/疲労/弾薬/戦線維持を更新
  -> 勝敗判定
```

## 次段階: 戦線指定と自律行動

現行の最寄り目標射撃モデルは初期RTS処理として有効だが、製品版方向では直接操作RTSに寄せすぎない。

次段階では、戦術マップをより広くし、プレイヤーは各旅団に以下を指定する。

- 担当戦線/線区
- 基準位置/anchor
- その周辺で自律行動する半径
- 目標優先順位
- 後退条件
- 使用/防衛/修理する施設

これにより、塹壕、バリケード、補給所、観測所、後退線が「背景」ではなく、旅団の自律判断を形作る戦術要素になる。

設計詳細は `docs/10_TACTICAL_AUTONOMY_AND_FRONTLINE.md` に移した。

## 武器差

初期値はゲームバランス仮値。実プレイ感に応じて調整する。

| 装備 | 役割 | 射程 | 火力 | 発射頻度 |
| --- | --- | ---: | ---: | ---: |
| 針撃銃 | 標準歩兵の主戦列火力 | 30 | 1.0 | 1.0 |
| 猟兵銃 | 長射程の散兵/森林戦 | 42 | 0.9 | 0.85 |
| 野戦砲 | 長射程支援、弾薬負担大 | 64 | 2.8 | 0.48 |
| 工兵器材 | 近距離自衛、築城/修理優先 | 16 | 0.45 | 0.65 |

## 敵軍モデル

敵は無限湧きの背景ではなく、波ごとの軍単位として動く。

| 敵 | 初期役割 | 挙動 |
| --- | --- | --- |
| Undead Mob | 数量圧、近接圧力 | 高速で接近し、近距離で損害を与える |
| Undead Riflemen | 遠距離圧 | 遅いが射程内から削る |
| Brute | 戦線破壊 | 近距離で士気/損耗圧を高める |
| Undead Officer | 指揮脅威 | 後続波で出現する優先目標 |

## 防衛施設モデル

防衛施設は戦闘画面上に座標を持ち、戦場処理にも参加する。

| 施設 | 初期役割 |
| --- | --- |
| Trench | 強い遮蔽、敵減速、施設火力、防衛線の中核 |
| Barricade | 近距離障害、局地的な敵減速 |
| Supply Depot | 弾薬回復支援、弱い施設火力、守るべき後方施設 |

工兵の `Build / Repair` 命令は、近くの損耗施設を修理する。敵が施設近くまで来ると耐久が減り、0になると `overrun` になる。

## 残課題

- 地形タグによる射程/移動/遮蔽補正はまだ薄い。
- 射線遮断、丘の高低差、森林視界、橋/隘路の通行制御は未実装。
- 部隊幅、隊列回頭、側面/背面判定はまだ抽象。
- 敵将校のバフ範囲や、Bruteの突破行動は今後の調整対象。
- プレイヤーの直接ドラッグ移動/配置変更は未実装。現状は命令ボタンによる自動移動。
- Standing Order / Frontline Assignmentは未実装。現状は即時命令と最寄り目標ロジックが中心。
