# CLAUDE.md — Takawasi Game goal loop入口(v2.7.1)

このファイルはClaude Code / Coworkがセッション開始時に自動で読む(ターンごとではない)。
役割は2つ**だけ**: ①モデルの癖への矯正(常駐でしか効かない反癖規則) ②番号MDを歩く
フローの配線。内容・手順の本体は番号MDが持つ。**ここに機構の要約を足さない**
(常駐の肥大は矯正カーネルを薄める。順癖の規則はファイルに置いても発火する——
常駐予算は訓練既定と戦う規則にだけ使う)。
テンプレをサブフォルダ(例: `loop/`)に置いた場合はパスを実配置に読み替えること。

## 矯正カーネル(モデルの訓練既定への上書き。病名は11_GOVERNANCE病理図鑑)

1. バッチ・フェーズ境界で「続けてよいか」と**聞かない**。質問は答えが次の行動を
   変えるときだけ。運転嗜好(00_MISSION)が継続なら、境界では報告して続行する【許可待ち病】
2. 「直した/できた」を、対応する新しい証跡(テスト・スクショ・実行ログ)なしに**書かない**。
   証跡が出せないなら「未検証」と書く【証跡なし完了病】
3. 「できない」と判定する前に、最小プローブを試すか07_KNOWLEDGE能力台帳の負の知識を引く。
   訓練知識由来の不可能判定で目標・完了条件を切り下げない【化石限界病】
4. 検証の厳密さは段階と不可逆性に比例させる——MVP前の探索に磨き段階の検証を要求しない。
   逆に、不可逆操作(削除・公開・送信・マージ)は体感の軽さに関係なく停止して人間確認
   【段階不一致検証病/不可逆境界】
5. 構造の欠陥に気づいたら、その場のパッチでなく規則ファイル・テンプレ側を直す
   (即改訂できないなら09_LEARNINGSに積む)。同型の修正2回目は原本改訂が義務【下流貼り紙病】
6. (Claude系のみ)画像アセットの生成はCODEX CLIへ委譲する——司令書(参照画像・雰囲気統一・
   受入基準)をまとめて `codex exec` に読ませ、CODEXの裁量で組み込みimage_genに生成させる。
   Claudeの役割は司令書作成と目視検収まで。「画像が作れないから」とコード描画(SVG/PIL等)で
   それらしく自作代替**しない**(例外: 図表・プロット等の決定的描画か人間の明示指示)【自作代替病】

## フロー配線(番号MDのライフサイクル。番号は読み順でなく**発火時期の順**)

| 時期 | 従うMD | やること |

| 毎セッション開始 | START.md起動手順 | `bash tools/session_brief.sh`(状態・計器・配車・現在番の規則が注がれる。手で集めない) |

## プロジェクト固有の柵

> このプロジェクトは、戦略・戦術・軍団育成・防御陣地の判断が相互に履歴を残す、製品版規模の長期戦術キャンペーンゲームを完成させるために存在する。

- 製品版/DLC1構造をMVPへ縮小しない。戦略/戦術mapを分離し、主戦場は省略不可、小作戦のみauto-resolve可。
- logic-firstと戦術結果の後続画面carryoverを優先する。実機QA済み/未確認を混同しない。
- 金銭・認証・契約・公開・送信・merge等の不可逆操作は人間確認まで停止する。

## 既存プロジェクトから引き継いだ規則

- 非自明な変更は親 `../FILEMAP.md` へ予定touch・責務・検証を先行登録する。
- 大きな実装はforeground CCDSへwork-orderで委譲し、指揮側が差分・FILEMAP・実機QAを最終確認する。
- QAは一巡→P0/P1/P2記録→batch修正→同導線再確認。mobile固有QAは現行ゲート外。
