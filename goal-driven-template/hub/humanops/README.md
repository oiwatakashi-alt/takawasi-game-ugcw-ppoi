# humanops — 人間操作キュー

エージェントが代行できない/してはいけない操作の受け渡し口。
起票→ダッシュボード(`DASHBOARD.html`)再生成→人間がクリック行脚→エージェントが完了確認、で回す。

## エスカレーション梯子(起票の前に必ず順に踏む)

1. **自動**: API・CLIで処理できるか(できるならやる。ここで終わり)
2. **ブラウザ代行**: Cowork(Claude in Chrome / computer use)で操作を代行できるか。
   Codex単騎で不可なら `hub/mail/to-cowork/` に依頼してCoworkに試させる
3. **人間専任**: 上記で不可、または下記リストに該当 → ここに起票

## 人間専任リスト(梯子に関係なく必ず人間。エージェントは代行禁止)

- 金銭の移動・課金・決済・サブスク契約(カード入力含む)
- 公的機関の本人認証・行政手続き・法的同意(利用規約の実質的な承諾を含む)
- サーバー・ドメイン等の契約締結
- 2FA・SMS認証などの本人性確認

## 起票フォーマット(`open/YYYY-MM-DD_件名.md`)

```
# [件名: 何をしてもらうか一行]
- 種別: [課金/認証/契約/行政/その他]
- なぜ必要: [どのマイルストーンが止まっているか]
- 一次リンク: [公式URL。手続きの開始地点に直行するもの]
- 手順: [1.クリック順で 2.具体的に 3.迷わない粒度で]
- 事前情報: [入力に使う値。秘匿情報は値でなく所在参照(接続情報.md等)で書く]
- 完了確認: [完了をエージェントがどう機械的に検証するか(例: APIが200を返す/DNSが引ける)]
- 期限/緊急度: [あれば]
```

## 運用

- 起票したら `python3 tools/dashboard_html.py` を実行して `DASHBOARD.html` を再生成し、
  04_STATEのブロッカー欄に記す。該当マイルストーンはブロック扱い
- 人間は `DASHBOARD.html` を開いてリンクから手続きする(見た目より一次リンクの正確さ優先。
  手続きに加えてユーザー資料の状態・反証・現在地も同じ1枚で確認できる)
- エージェントは次セッションの偵察番で「完了確認」欄の方法を実行し、
  通ったら該当ファイルを `done/` に移してダッシュボードを再生成する
- **棚上げが既定**: 起票してもループは止めない。進められるマイルストーンがある限り進み、
  デプロイ可能な状態まで自走で近づける(計画の射程はデプロイ展開まで——01_PLAN既定)
- **ゴール条件との締結**: `open/` が空でない限りフェーズの完了条件は満たされない
  (02_GOAL.mdの雛形に組込済み)。全マイルストーンが人間待ちになったときのみ
  fail-stop(整備番と発見リスト調査のみ可)
- 秘匿情報(パスワード・トークン・カード番号)は起票ファイルにもHTMLにも**書かない**。
  所在参照のみ
