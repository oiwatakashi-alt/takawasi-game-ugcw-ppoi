# 学びの台帳 — フロー自体の改善(二階のループの入口)

対象は**やり方**の学びであって、プロダクトの気づきではない
(プロダクトの気づき→04_STATEの発見リスト。ここに書くのは「この手順は迂遠だった」
「この規則が邪魔をした」「こうしたら速かった」の類)。

規則:
- 気づいたターンに1行で追記する(その場で規則を直さない——直すのは刈り取り時)
- **刈り取りは監査番の仕事**(03_DISPATCH監査番の規則7): 未振り分けの行を3つに振り分ける
  1. プロジェクト限り → このプロジェクトのコピー(00/03等)を直接改訂し、改訂した旨をここに記す
  2. テンプレ候補 → 汎用性があるもの。テンプレ本体のCHANGELOG改版候補として印を付け、
     人間チェックポイントで提示する(テンプレ本体の改訂は人間の承認後)
  3. 保留 → 判断材料不足。次の刈り取りへ
- 同じ学びが2つの異なるプロジェクトで出たら、プロジェクトでなくテンプレの問題と推定する
  (11_GOVERNANCE 第六命題)
- 改版は観測された失敗・成功を根拠にのみ行う。予防的な規則追加はここでもしない

## 記入形式

```
- [日付/番] 学び: [一行] / 提案: [あれば] / 振り分け: [未 / プロジェクト限り(改訂済) / テンプレ候補 / 保留]
```
