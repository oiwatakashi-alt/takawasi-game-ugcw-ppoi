# AGENTS.md — Takawasi Game入口(goal-driven-template v2.7.1・Codex用)

このファイルはCodexが起動時に一度だけ読む(32KiB上限に注意して薄く保つ)。
役割は2つ**だけ**: ①モデルの癖への矯正(常駐でしか効かない反癖規則) ②番号MDを歩く
フローの配線。内容・手順の本体は番号MDが持つ。**ここに機構の要約を足さない**
(常駐の肥大は矯正カーネルを薄める。順癖の規則はファイルに置いても発火する——
常駐予算は訓練既定と戦う規則にだけ使う)。
テンプレをサブフォルダ(例: `loop/`)に置いた場合はパスを実配置に読み替えること。

## 実配置

- ループ本体は `goal-driven-template/`。番号MD・`START.md`・`tools/`・`hub/`・`plans/`への参照は全てこの接頭辞で解決する。
- セッション開始は `bash goal-driven-template/tools/session_brief.sh`。唯一の入口は `goal-driven-template/START.md`。

## 矯正カーネル(モデルの訓練既定への上書き。病名は11_GOVERNANCE病理図鑑)

1. バッチ・フェーズ境界で「続けてよいか」と**聞かない**。質問は答えが次の行動を
   変えるときだけ。運転嗜好(00_MISSION)が継続なら、境界では報告して続行する【許可待ち病】
2. 「直した/できた」を、対応する新しい証跡(テスト・スクショ・実行ログ)なしに**書かない**。
   証跡が出せないなら「未検証」と書く【証跡なし完了病】
3. 「できない」と判定する前に、最小プローブを試すか07_KNOWLEDGE能力台帳の負の知識を引く。
   訓練知識由来の不可能判定で目標・完了条件を切り下げない【化石限界病】
4. 検証の厳密さは段階と不可逆性に比例させる——MVP前の探索に磨き段階の検証を要求しない。
   逆に、不可逆操作(削除・公開・送信・マージ)は体感の軽さに関係なく停止して人間確認
   【段階不一致検証病/不可逆境界】
5. 構造の欠陥に気づいたら、その場のパッチでなく規則ファイル・テンプレ側を直す
   (即改訂できないなら09_LEARNINGSに積む)。同型の修正2回目は原本改訂が義務【下流貼り紙病】

## フロー配線(番号MDのライフサイクル。番号は読み順でなく**発火時期の順**)

| 時期 | 従うMD | やること |
|---|---|---|
| 初回セットアップ | START.md → 00→01→02→04 | 順に埋める(下書きはエージェント、確定は人間) |
| 毎セッション開始 | START.md起動手順 | `bash tools/session_brief.sh`(状態・計器・配車・現在番の規則が注がれる。手で集めない) |
| 毎ターン | 03(注入された分)+04+08 | 配車を引用→番の規則だけで作業→commit `[番名] 要旨`→STATE欄置換→LOG追記(20行以内)→`bash tools/turn_close.sh`(赤はamend) |
| フェーズ境界 | 02→01→00の逆順照合 | 完了判定(`bash tools/stop_check.sh`の棚卸しを引用)→遷移儀式→**三窓レビュー盤を起動**(ブリーフ実体: `hub/briefs/review_macro.md` `review_micro.md` `review_market.md`。{ }を埋めて渡すだけ)→02書き直し(北極星の逐語引用必須) |
| 発火時のみ | 05委譲契約/06レビュー計数/07調べ物・着任パック/09学び/10命名/11人間側管理 | 状況の発火条件で開く(地図はSTART.md) |

検証が失敗したまま先に進まない(stop-and-fix)。委譲するか直営かは01_PLANの
実行者フィールドが決める。サブエージェント分離が使えない場合は単騎モード(05_CONTRACTS.md末尾)。

## プロジェクト固有の柵

> このプロジェクトは、戦略・戦術・軍団育成・防御陣地の判断が相互に履歴を残す、製品版規模の長期戦術キャンペーンゲームを完成させるために存在する。

- 製品版/DLC1までの構造を前提とし、MVP・prototypeへ縮小しない。
- 戦略キャンペーンマップと戦術バトルマップを分離する。主戦場は省略不可、3〜4件の小作戦だけauto-resolve可。
- 戦術ロジックと戦果の後続画面への持越しを先に成立させ、アートは識別性に効く段階で入れる。
- 実機QAの確認済み/未確認を混同しない。`FILEMAP.md` の状態語と実挙動を一致させる。
- 金銭・認証・契約・公開・外部送信・merge等の不可逆操作は人間確認まで停止する。

## 既存プロジェクトから引き継いだ規則

- 非自明な変更は `FILEMAP.md` へ予定touch・責務・検証を先行登録する。
- 大きな実装はDS V4 / CCDSへ明示的なwork-orderで委譲し、CCDSはforeground起動。Codexは差分・FILEMAP・実機QA・最終ゲートを担う。
- QAは対象導線を一巡してP0/P1/P2をまとめ、batch修正後に同じ導線を再確認する。
- 現在の製品QA対象はdesktop。mobile固有QAは再昇格するまで現行ゲート外。
